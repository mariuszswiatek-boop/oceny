## Szybki start i orientacja
Jeśli zaczynasz:
- Przeczytaj `docs/overview.md` i `docs/architecture.md`.
- Uruchom lokalnie według `docs/development.md`.
- Skonfiguruj środowiska wg `docs/configuration.md`.

## Mapa dokumentacji
- `docs/overview.md` — mapa systemu i komunikacja komponentów.
- `docs/development.md` — uruchomienie lokalne, debug, seed, lint.
- `docs/configuration.md` — wszystkie zmienne środowiskowe i profile.
- `docs/architecture.md` — architektura, diagramy, model danych.
- `docs/api.md` — kontrakty API i przykłady wywołań.
- `docs/cicd.md` — pipeline, release, deploy.
- `docs/deploy.md` — wdrożenie produkcyjne (Oracle + Docker).
- `docs/runbook.md` — operacje i procedury.
- `docs/troubleshooting.md` — objawy i rozwiązania.
- `docs/security.md` — bezpieczeństwo i rekomendacje.
- `docs/adr/README.md` — rejestr decyzji architektonicznych.
- `README.md` — skrót wprowadzający i linki.

## Jak zgłaszać braki
- Jeśli w dokumentacji jest TODO/Wymaga potwierdzenia, sprawdź wskazaną ścieżkę pliku lub uruchom wskazane polecenie i uzupełnij opis.

## Jak zweryfikować
- Otwórz wszystkie linki z sekcji „Mapa dokumentacji” i sprawdź, czy prowadzą do istniejących plików.
